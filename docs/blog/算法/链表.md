::: tip **链表（Linked List）**：一种线性表数据结构，通过一组任意（可连续或不连续）的存储单元，存储同类型数据。
:::
在链表结构中，数据元素之间的逻辑顺序由指针维护。虽然逻辑上相邻的数据元素在物理内存中可以相邻，也可以完全不相邻，因此链表在物理存储上的分布是非连续、随机的。

链表的优缺点如下：

- **优点**：链表无需预先分配存储空间，按需动态申请，能够有效避免空间浪费；在插入、删除等操作上，链表通常比数组更高效，尤其是在需要频繁修改数据结构时表现突出。
- **缺点**：链表除了存储数据本身外，还需额外存储指针信息，因此整体空间开销大于数组；同时，链表不支持随机访问，查找元素时需要从头遍历，效率较低。

### 链表类型

| 类型         | 特点                     | 适用场景     |
| ------------ | ------------------------ | ------------ |
| **单链表**   | 每个节点只有一个后继指针 | 基础链表操作 |
| **双向链表** | 每个节点有前驱和后继指针 | 需要双向遍历 |
| **循环链表** | 尾节点指向头节点形成环   | 循环访问场景 |

### 基本操作复杂度

| 操作     | 时间复杂度   | 空间复杂度 | 说明                   |
| -------- | ------------ | ---------- | ---------------------- |
| **查找** | O(n)*O*(*n*) | O(1)*O*(1) | 需要遍历到目标位置     |
| **插入** | O(n)*O*(*n*) | O(1)*O*(1) | 找到插入位置后操作简单 |
| **删除** | O(n)*O*(*n*) | O(1)*O*(1) | 找到删除位置后操作简单 |
| **修改** | O(n)*O*(*n*) | O(1)*O*(1) | 需要遍历到目标位置     |

### 链表 vs 数组

| 特性         | 链表                         | 数组                       |
| ------------ | ---------------------------- | -------------------------- |
| **存储方式** | 链式存储，非连续             | 顺序存储，连续             |
| **随机访问** | 不支持，O(n)*O*(*n*)         | 支持，O(1)*O*(1)           |
| **插入删除** | 高效，O(1)*O*(1)（已知位置） | 需要移动元素，O(n)*O*(*n*) |
| **空间开销** | 额外指针开销                 | 无额外开销                 |
| **内存分配** | 动态分配                     | 静态分配                   |

### 应用场景

- **频繁插入删除**：链表在插入删除操作上比数组更高效
- **动态内存管理**：适合内存大小不确定的场景
- **实现其他数据结构**：栈、队列、哈希表等的基础
- **算法优化**：某些算法中链表结构能提供更好的性能

### JAVA的LinkedList

链表由若干链节点通过 next 指针依次连接而成。通常我们会先定义一个简单的「链节点类」，再基于此实现完整的「链表类」。

- **链节点类（ListNode）**：包含成员变量 val（存储数据元素的值）和 next（指向下一个节点的指针）。
- **链表类（LinkedList）**：包含一个链节点变量 head，用于表示链表的头节点。

**Java中链节点与链表结构的代码实现如下：**

```java
/**
 * LinkedList的 Node 节点通过 prev 和 next 指针来连接。这些引用本质上是JVM的内存地址，只在程序运行时有效。
 * 直接序列化它们不仅没有意义，反序列化时还会因为地址失效导致整个链表结构损坏，所以用transient 修饰。
*/

public class LinkedList<E>
    extends AbstractSequentialList<E>
    implements List<E>, Deque<E>, Cloneable, java.io.Serializable
{
	// [!code word:transient]
    transient int size = 0;

    /**
     * Pointer to first node.
     */
    transient Node<E> first; 

    /**
     * Pointer to last node.
     */
    transient Node<E> last;
    

    //链节点类,定义节点属性
    private static class Node<E> {
        //节点值
        E item;
        //头指针
        Node<E> next;
        //尾指针
        Node<E> prev;

        Node(Node<E> prev, E element, Node<E> next) {
            this.item = element;
            this.next = next;
            this.prev = prev;
        }
    }
}
```

**Java中创建链表代码实现如下**

通过一次 `toArray()` 获取所有待插入元素，避免了在循环中反复调用集合迭代器；通过定位前驱和后继节点，利用循环在二者之间“搭桥”建立新节点链，最后连接断开的原有部分。这种实现保证了插入操作的**时间复杂度为 O(n + numNew)**（其中 n 为到达插入位置的遍历开销），空间复杂度仅与插入元素数量相关。

```java
    public boolean addAll(int index, Collection<? extends E> c) {
        //检查index是否超出范围，若超出范围会 IndexOutOfBoundsException
        checkPositionIndex(index);
		//将传入的集合转换为 Object 数组。在后续循环中可以直接访问元素，避免迭代器在修改过程中的并发修改问题，并且可以预先知道元素个数。
        Object[] a = c.toArray();
        int numNew = a.length;
        if (numNew == 0)
            return false;
		//声明两个 Node 引用,pred：插入位置的前驱节点(新元素将接在它后面),succ：插入位置的后继节点(原 index 位置的节点，新元素将插在它前面)
        Node<E> pred, succ;
        //当 index 等于当前链表长度时，表示在末尾追加
        if (index == size) {
            //末尾没有后继节点。
            succ = null;
            //前驱节点就是当前链表的尾节点 last。
            pred = last;
        //当 index != size 时，表示在中间或头部插入
        } else {
            //调用 node(index) 方法，通过遍历找到当前位于 index 位置的节点，作为后继节点 succ。
            // [!code word:node]
            succ = node(index);
            //前驱节点就是 succ 的前一个节点。
            pred = succ.prev;
        }
		//遍历数组中的每一个元素。
        for (Object o : a) {
            //由于 c.toArray() 返回的是 Object[]，这里需要强转为泛型类型 E。@SuppressWarnings 用于抑制编译器关于未经检查类型转换的警告。
            @SuppressWarnings("unchecked") E e = (E) o;
            //创建一个新节点，构造函数参数为 (前驱节点, 元素数据, 后继节点)。此时后继节点先设为 null，因为后续循环会不断调整 pred 的指向。
            Node<E> newNode = new Node<>(pred, e, null);
            //如果 pred 为 null，说明当前插入的是链表的第一个节点（即 index == 0 且链表原本为空，或是从头部开始插入的第一个元素）。
            if (pred == null)
                //将链表头指针 first 指向这个新节点。
                first = newNode;
             //否则，说明有前驱节点。
            else
                //将前驱节点的 next 指针指向新节点，完成前向连接。
                pred.next = newNode;
            //将 pred 引用更新为刚刚插入的新节点，为下一轮循环做准备。这样新插入的节点就成为了后续插入节点的前驱，从而保持连续。
            pred = newNode;
        }
		// 如果 succ 为 null，说明是在链表末尾插入，此时最后一个新插入的节点就是新的尾节点。
        if (succ == null) {
            //last = pred：将链表尾指针 last 更新为最后一个新节点（即循环结束时的 pred）。
            last = pred;
        //否则，说明插入位置后方还有原有节点（succ）。    
        } else {
            //将最后一个新节点的 next 指针指向 succ。
            pred.next = succ;
            //将 succ 的 prev 指针指回最后一个新节点，完成双向连接。
            succ.prev = pred;
        }
		//更新链表元素总数。
        size += numNew;
        //修改计数器自增，用于实现 fail-fast 机制。当在迭代过程中链表结构发生变化时，迭代器能通过检查此计数抛出 ConcurrentModificationException。
        modCount++;
        //返回 true，表示链表已修改（除非集合为空，前面已经提前返回 false）。
        return true;
    }


//node(int index) 是 LinkedList 的核心私有方法，用于根据索引快速定位到链表中的节点。它的设计巧妙利用了双向链表的特性，将遍历次数减少了一半。
Node<E> node(int index) {
        //这是一个断言，仅在开发调试时有效，用于确认传入的 index 是有效元素索引（即 0 <= index < size）。在生产环境通常被禁用。
        // assert isElementIndex(index);
		//这是位运算右移一位，等价于数学上的 size / 2，但执行速度稍快。这行代码判断目标索引是否位于链表的前半部分。
        if (index < (size >> 1)) {
            //若在前半部分：则从头节点 (first) 开始，向后遍历 index 次，找到目标节点并返回。
            Node<E> x = first;
            for (int i = 0; i < index; i++)
                x = x.next;
            return x;
            //若在后半部分：则从尾节点 (last) 开始，向前遍历 (size - 1) - index 次，找到目标节点并返回。
        } else {
            Node<E> x = last;
            for (int i = size - 1; i > index; i--)
                x = x.prev;
            return x;
        }
    }
```

**查找节点**

两种查找indexOf和lastIndexOf 都是遍历列表内容然后equals 没什么好说的。

```java
//返回该列表中指定元素第一次出现的索引，如果列表中不包含该元素，则返回 -1。    
public int indexOf(Object o) {
        int index = 0;
        if (o == null) {
            for (Node<E> x = first; x != null; x = x.next) {
                if (x.item == null)
                    return index;
                index++;
            }
        } else {
            for (Node<E> x = first; x != null; x = x.next) {
                if (o.equals(x.item))
                    return index;
                index++;
            }
        }
        return -1;
    } 


//返回该列表中指定元素最后一次出现的索引，如果列表中不包含该元素，则返回 -1。    
public int lastIndexOf(Object o) {
        int index = size;
        if (o == null) {
            for (Node<E> x = last; x != null; x = x.prev) {
                index--;
                if (x.item == null)
                    return index;
            }
        } else {
            for (Node<E> x = last; x != null; x = x.prev) {
                index--;
                if (o.equals(x.item))
                    return index;
            }
        }
        return -1;
    }
```

**插入节点**

```java
    
public void add(int index, E element) {
        //检查index是否越界
        checkPositionIndex(index);
		//index等于列表长度直接在尾部添加
        if (index == size)
            linkLast(element);
        else
            //index != size在 前面添加,此处仍调用node方法去查找对应index位置的节点信息
            linkBefore(element, node(index));
}

void linkLast(E e) {
    	//尾节点变量
        final Node<E> l = last;
    	//定义我们要添加的节点
        final Node<E> newNode = new Node<>(l, e, null);
    	//直接让last类变量等于 newNode
        last = newNode;
    	//尾节点等于空，说明列表中没有元素
        if (l == null)
            //此时让头节点也等于我们的新节点
            first = newNode;
        else
            //尾节点的下一指针指向我们的新节点
            l.next = newNode;
        size++;
        modCount++;
}
  
void linkBefore(E e, Node<E> succ) {
        // assert succ != null;
    	//定义pred变量，值为前面通过node方法获取的节点信息
        final Node<E> pred = succ.prev;
    	//定义新节点
        final Node<E> newNode = new Node<>(pred, e, succ);
    	//让之前的节点的头节点指向我们的新节点
        succ.prev = newNode;
    	//如果直接的头节点为空，说明链表中没有元素，直接将我们的新节点设置为头结点
        if (pred == null)
            first = newNode;
        else
            //之前的头节点不等于空，将之前头结点的下一指针指向我们的新节点
            pred.next = newNode;
        size++;
        modCount++;
}
```

**改变节点**

```java
//将列表中指定位置的元素替换为指定的元素。
public E set(int index, E element) {
    	//检查index是否越界
        checkElementIndex(index);
    	//调用node获取对应index节点信息
        Node<E> x = node(index);
    	//获取旧值
        E oldVal = x.item;
    	//为节点赋新值
        x.item = element;
    	//返回旧值
        return oldVal;
}
```

**删除节点**

```java
    
public E remove(int index) {
    //检查index是否越界
    checkElementIndex(index);
    //使用node获取对应index节点信息作为参数调用 unlink
    return unlink(node(index));
}

   E unlink(Node<E> x) {
        // assert x != null;
       //待删除节点
        final E element = x.item;
       //待删除节点下一节点
        final Node<E> next = x.next;
       //待删除节点上一节点
        final Node<E> prev = x.prev;
		//如果待删除节点上一节点等于空,说明是第一个元素，就让待删除节点下一节点等于头结点
        if (prev == null) {
            first = next;
        } else {
            //如果待删除节点上一节点不等于空，就让待删除节点上一节点的尾指针指向待删除节点下一节点
            prev.next = next;
            //将待删除节点头指针置空
            x.prev = null;
        }
		//如果待删除节点下一节点等于空,说明是最后一个元素，就让待删除节点下一节点等于尾结点
        if (next == null) {
            last = prev;
        } else {
            //如果待删除节点下一节点不等于空，就让待删除节点下一节点的头指针指向待删除节点上一节点
            next.prev = prev;
            //将待删除节点尾指针置空
            x.next = null;
        }
        x.item = null;
        size--;
        modCount++;
        return element;
    }
```
